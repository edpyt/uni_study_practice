# Аналитический обзор выполнения кейс-задачи №2 "Счетчик"

## 1. Функциональность

### Достоинства

В ходе выполнения работы мне удалось полностью реализовать базовый функционал, требуемый в задании:

- **Пользовательский интерфейс** - создана страница с двумя кнопками ("плюс" и "минус") и полем отображения результата
- **Стилизация** - применен линейный градиент для фона страницы, градиент для кнопок, выполнено скругление углов, элементы отцентрированы относительно окна браузера
- **Логика счетчика** - при нажатии на кнопки значение увеличивается или уменьшается на единицу
- **Динамическое изменение цвета** - фон поля результата меняется в зависимости от значения
- **Блокировка кнопок** - при достижении значения 10 кнопка "плюс" деактивируется, при достижении -10 кнопка "минус" деактивируется
- **Информационное сообщение** - при достижении экстремальных значений (±10) появляется предупреждение

### Недостатки
- Часть стилей прописана инлайн в атрибутах, часть - в `<style>`
- Граничные значения (10, -10) и цвета не вынесены в константы

#### Способы решения
- Все инлайн-стили необходимо вынести в CSS-классы:
```html
<!-- Было: -->
<div style="display: flex; width: 100%; justify-content: space-between;">

<!-- Стало: -->
<div class="button-container">
```

```css
/* Добавить в <style> */
.button-container {
display: flex;
width: 100%;
justify-content: space-between;
}
```

- Вынести все магические числа и цвета в константы в начале скрипта:
```js
// Добавить в начало скрипта:
const COUNTER_CONFIG = {
  MAX_VALUE: 10,
  MIN_VALUE: -10,
  STEP: 1,
  COLORS: {
    POSITIVE: 'yellow',
    NEGATIVE: 'green',
    ZERO: 'red'
  }
};

// Использование:
// Было: if (inputResult.value >= 10)
// Стало: if (inputResult.value >= COUNTER_CONFIG.MAX_VALUE)
```

## 4. Безопасность
### Достоинства
- Базовая защита реализована через атрибуты min/max и disabled в HTML

### Недостатки
- Отсутствие валидации входных данных - через консоль разработчика можно установить любое значение
- Возможность обхода блокировки кнопок через инспектор браузера
- Нет защиты от установки нечисловых значений
- Отсутствует экранирование данных

#### Способы решения
- Добавить проверку значения при каждом обновлении и защиту от NaN
```javascript
function validateAndUpdate(op) {
  let currentValue = Number(inputResult.value);
  if (isNaN(currentValue)) {
    currentValue = 0;
    inputResult.value = 0;
  }

  let newValue = op === '+' ? currentValue + 1 : currentValue - 1;
  newValue = Math.max(-10, Math.min(10, newValue));

  inputResult.value = newValue;
}
```
